## 更新类别

### 🚀 新功能
平台新增的主要功能和能力。

### 🔧 改进
现有功能的增强、性能优化和用户体验升级。

### 🐛 错误修复
已知问题的解决和稳定性改进。

### ⚠️ 重大变更
可能需要修改集成代码的更新。

### 🔒 安全
安全补丁和漏洞修复。

## 发布计划

我们遵循定期发布计划以确保持续改进：

| 发布类型 | 频率 | 描述 |
|---------|------|------|
| **主要版本** | 季度 | 重大新功能和破坏性变更 |
| **次要版本** | 月度 | 新功能和改进 |
| **补丁版本** | 周度 | 错误修复和安全更新 |
| **热修复** | 按需 | 关键安全和稳定性修复 |

<Note>
所有发布都会在此记录，并在部署前 48 小时通过电子邮件通知注册用户。
</Note>

## 迁移指南

当我们发布破坏性变更时，我们会提供全面的迁移指南：

<Steps>
  <Step title="查看变更">
    检查发布说明中影响您集成的破坏性变更
  </Step>
  
  <Step title="在沙箱测试">
    使用我们的沙箱环境测试您的代码与新版本的兼容性
  </Step>
  
  <Step title="更新代码">
    按照我们的迁移指南更新您的实现
  </Step>
  
  <Step title="部署">
    在弃用截止日期之前部署更新后的代码
  </Step>
</Steps>

<Warning>
API v1 将于 **2025 年 1 月 31 日** 停用。请在此日期之前迁移到 API v2。详见我们的[迁移指南](/zh/api-reference/introduction)。
</Warning>
